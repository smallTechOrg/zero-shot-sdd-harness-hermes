# API — `scaffold-agent`

> Run commands are exact. Stubs fail gracefully; live path passes when a key is in `.env`.

---

## API Style

REST (FastAPI). Two routes in Phase 1, plus optional `/docs` auto-generated by FastAPI.

## Endpoints / Commands

### `GET /health`

**Purpose:** Liveness and library availability check.

**Response:**
```json
{
  "status": "ok",
  "version": "0.1.0"
}
```

**Error cases:**
| Status | Condition |
|--------|-----------|
| 500 | SQLAlchemy engine init fails. |

---

### `POST /api/chat`

**Purpose:** Primary agent entry point. Accepts a chat message and returns an assistant reply.

**Request:**
```json
{
  "messages": [
    { "role": "user", "content": "Hello" }
  ]
}
```

**Response (200 OK):**
```json
{
  "reply": "Echo: Hello (stub response — set LLM_API_KEY for live model)"
}
```

When `LLM_API_KEY` is present and `LLM_BASE_URL` is set, real integration calls the configured endpoint and returns the model output unchanged.

**Error cases:**
| Status | Condition |
|--------|-----------|
| 400 | `messages` missing, not a list, or first item malformed. |
| 422 | Validation failure (Pydantic). |
| 500 | Internal error in agent stub (logged with traceback to stdout). |

## Notes

- Auth is disabled in Phase 1; CORS is open for local dev. Production deploy should set CORS origins.
- Streaming is a Phase 2 enhancement.
